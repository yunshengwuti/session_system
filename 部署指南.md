# 免费部署指南

## 方案

前端部署到 Vercel，后端部署到 Render，数据库使用 Render PostgreSQL。

这套配置适合当前项目：
- 前端：Vue + Vite
- 后端：FastAPI
- 数据库：本地仍可用 MySQL，线上使用 Render 提供的 `DATABASE_URL` 连接 PostgreSQL

---

## 第一步：确认分支

当前部署代码应该在 `v2.0` 分支提交：

```bash
git branch --show-current
```

如果不是 `v2.0`，先切换：

```bash
git switch v2.0
```

---

## 第二步：提交并推送代码

确认改动：

```bash
git status
git diff
```

提交到本地：

```bash
git add backend/config/database.py backend/requirements.txt frontend/src/api/index.js frontend/vercel.json 部署指南.md
git commit -m "Prepare free deployment"
```

推送到远程 `v2.0`：

```bash
git push origin v2.0
```

---

## 第三步：部署数据库到 Render

1. 打开 https://render.com
2. 用 GitHub 登录
3. 点击 **New +**，选择 **PostgreSQL**
4. 推荐配置：
   - Name: `session-system-db`
   - Database: `session_system`
   - User: `session_user`
   - Region: `Singapore`
5. 创建完成后，进入数据库详情页
6. 复制 **Internal Database URL**

后端服务和数据库都在 Render 时，优先使用 Internal Database URL。

---

## 第四步：部署后端到 Render

1. 点击 **New +**，选择 **Web Service**
2. 连接 GitHub 仓库
3. 选择 `v2.0` 分支
4. 配置：
   - Name: `session-system-backend`
   - Root Directory: `backend`
   - Runtime: `Python 3`
   - Build Command: `pip install -r requirements.txt`
   - Start Command: `uvicorn main:app --host 0.0.0.0 --port $PORT`

添加环境变量：

```text
DATABASE_URL=你的 Render PostgreSQL Internal Database URL
ZHIPU_API_KEY=你的智谱 API Key
```

如果暂时没有智谱 API Key，可以先不填 `ZHIPU_API_KEY`，但生成 AI 报告相关功能可能不可用。

部署完成后，复制后端地址，例如：

```text
https://session-system-backend.onrender.com
```

测试后端：

```text
https://session-system-backend.onrender.com/docs
```

---

## 第五步：部署前端到 Vercel

1. 打开 https://vercel.com
2. 用 GitHub 登录
3. 点击 **Add New**，选择 **Project**
4. 选择同一个 GitHub 仓库
5. 选择 `v2.0` 分支
6. 配置：
   - Framework Preset: `Vite`
   - Root Directory: `frontend`
   - Build Command: `npm run build`
   - Output Directory: `dist`

添加环境变量：

```text
VITE_API_BASE_URL=https://你的后端地址.onrender.com/api
```

示例：

```text
VITE_API_BASE_URL=https://session-system-backend.onrender.com/api
```

然后点击 **Deploy**。

---

## 第六步：测试

打开 Vercel 给你的前端地址，例如：

```text
https://session-system-xxx.vercel.app
```

测试顺序：

1. 访问首页是否正常
2. 上传 Excel 数据
3. 查看会话列表
4. 进入会话详情
5. 生成日报或周报

---

## 本地开发方式

本地不配置 `DATABASE_URL` 时，后端会继续读取这些 MySQL 环境变量：

```text
DB_HOST=localhost
DB_PORT=3306
DB_USER=root
DB_PASSWORD=你的MySQL密码
DB_NAME=session_system
```

本地前端如果不配置 `VITE_API_BASE_URL`，会默认请求：

```text
http://localhost:8000/api
```

---

## 注意事项

1. Render 免费 Web Service 长时间无访问会休眠，首次访问可能需要等待唤醒。
2. Render 免费 PostgreSQL 有容量和期限限制，请定期备份重要数据。
3. 前端部署后如果请求失败，优先检查 Vercel 环境变量 `VITE_API_BASE_URL` 是否包含 `/api`。
4. 后端部署失败时，优先打开 Render 日志，检查 `DATABASE_URL` 和依赖安装错误。
